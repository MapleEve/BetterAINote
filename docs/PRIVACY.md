# 隐私说明

## 适用对象

这篇面向准备自托管 BetterAINote 的用户、部署者和贡献者。它说明 BetterAINote 可能保存什么、哪些数据不应公开，以及连接第三方来源和服务时需要自己承担的边界。

## 当前状态

BetterAINote 当前是 `preview`，默认面向私有部署。隐私结果取决于：

- 你把实例部署在哪里。
- 你连接了哪些录音来源。
- 你是否启用本地音频归档。
- 你使用哪个私有转写或 AI 标题服务。
- 你如何备份、分享日志和提交 Issue。

## BetterAINote 可能保存什么

| 数据 | 示例 |
| --- | --- |
| 录音元数据 | 标题、来源、时长、开始时间、同步状态。 |
| 本地音频 | 来源可下载且启用归档时，保存到 `LOCAL_STORAGE_PATH`。 |
| 转写数据 | 原文、段落、说话人、词级时间和转写任务状态。 |
| 来源报告 | 来源逐字稿、摘要和脱敏后的来源详情可用状态。 |
| SQLite 数据库 | 设置、录音库、转写、说话人、词级时间等拆分数据库。 |
| 凭据元数据 | 数据源连接状态、服务配置和加密保存的凭据相关记录。 |
| 日志 | app / worker 在你部署环境中产生的运行日志。 |

## 本地 SQLite 和录音目录

SQLite 文件便于备份和迁移，但也可能包含私人录音标题、来源状态、转写文本、说话人名称和凭据相关记录。

建议：

- 保持数据库目录在你控制的本地磁盘或可信存储上。
- 数据库目录和录音归档目录一起备份。
- 限制运行用户和备份用户的文件系统权限。
- 不要把数据库文件、音频文件或完整转写上传到公开 Issue。

## Provider 凭据

Provider 凭据包括但不限于录音来源账号凭据、会话字段、转写服务密钥和 AI 标题服务密钥。

不要公开：

- cookie、bearer、刷新凭据、会话字段。
- 组织 ID、用户 ID、录音 ID、临时下载链接或签名 URL。
- VoScript、转写服务、AI 标题服务或对象存储密钥。
- 能还原账号状态的截图、录屏或网络抓包文件。

如果怀疑泄露，请在对应来源或服务中轮换凭据。

## 日志和 Issue 脱敏

公开反馈前，请删除或替换：

- 私人录音标题、会议内容、说话人姓名。
- 所有账号凭据、服务密钥和会话字段。
- 本地绝对路径、服务器名、内网域名和项目私有目录。
- 上游完整请求 / 响应、网络抓包文件、完整环境文件。

推荐只保留：

- BetterAINote commit 或版本。
- 操作步骤。
- 来源名称和站点模式。
- HTTP 状态、脱敏错误码、脱敏字段名。
- 最小日志片段。

## 第三方服务

BetterAINote 可以连接外部录音来源，也可以配置私有转写和 AI 标题服务。每个服务都有自己的数据处理规则。

如果你希望最大化本地控制：

- 优先使用自托管转写服务。
- 只在接受服务条款的情况下启用外部 AI 标题生成。
- 开启数据源前确认该账号允许导出或同步相关记录。
- 不把生产凭据复用到测试实例。

## 公开文档边界

公开文档、示例、测试和模板只能使用占位符和脱敏样例。不要加入本地部署值、真实来源详情、内部排查记录、完整转写或未公开计划。
